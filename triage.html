Tab 3 — Triage
The Triage queue is where unrouted Gong calls land. After each sync, calls that don't match a known routing alias appear here. Your job is to review each call and assign it to a customer folder.
ColumnsDate · Organizer · Title · AI-generated summary · Transcript size (dimmed = short transcript, likely not useful)
RoutingPress Enter on a selected call to open the routing overlay. Type to search existing customers or type a new name to create one. After routing you'll be asked for an optional alias — a phrase from the call title that will auto-route future calls from this customer.
TabsUnprocessed shows calls awaiting routing. Processed shows already-routed calls.
Tab 4 — Tech Stack
Shows per-customer tech stack intelligence extracted from Gong call transcripts by Claude AI.
Filetech_stack.md per customer folder
Built byRunning /update-tech-stack in Claude Code after routing calls
ExtractsCurrent monitoring tools · infrastructure (cloud/on-prem) · languages and frameworks · existing Grafana usage · integration context
Use forPre-call prep — understand what the customer is running and where Grafana fits before you join
Tab 5 — 3 Whys
Shows per-customer 3 Whys intelligence — the strategic buying narrative extracted from calls. The framework captures: Why Anything (what pain is driving evaluation), Why Now (the urgency or trigger), and Why Grafana (the specific fit or differentiator).
File3_whys_summary.md per customer folder
Built byRunning /update-3-whys in Claude Code after routing calls
ExtractsEvidence for each of the three questions with verbatim quotes from transcripts
Use forSharpen your pitch, understand deal momentum, and identify gaps in your discovery
Setup (first time)
pip install google-cloud-bigquery flask
Install Python dependencies
gcloud auth application-default login
Authenticate with GCP for BigQuery access
python3 gong_fetch.py --init-owners
Scaffold gong_owners.json — then edit it to add your team's Gong owner IDs
echo "127.0.0.1 oppbot.local" | sudo tee -a /etc/hosts
Add local hostname so oppbot.local resolves (one time per machine)
echo "ANTHROPIC_API_KEY=sk-..." > .env
Required for AI enrichment (tech stack, 3 Whys)
CLI reference
./triage.sh
Start the server and open oppbot.local in your browser
python3 gong_fetch.py --sync
Fetch latest calls from BigQuery and auto-route them
python3 gong_fetch.py --status
Show sync status and call counts
python3 gong_fetch.py --account
Show current account and data directory
python3 gong_fetch.py --nuke
Reset all fetched data (keeps config)
./install-launchd.sh
Install macOS LaunchAgent for daily auto-sync
